# 理解数据结构：数组 vs 链表

## 简介

数据结构是高效编程的基础。为用例选择正确的结构可以显著影响性能。

## 数组

### 特性
- **访问时间**: O(1) - 通过索引直接访问
- **插入/删除**: O(n) - 可能需要移动元素
- **内存**: 连续分配
- **缓存友好**: 是

### 优点
- 快速随机访问
- 内存高效（无指针）
- 缓存局部性好

### 缺点
- 固定大小（许多语言中）
- 中间插入/删除成本高
- 未充分利用时浪费内存

## 链表

### 特性
- **访问时间**: O(n) - 必须从头遍历
- **插入/删除**: O(1) - 一旦找到位置
- **内存**: 非连续分配
- **缓存友好**: 否

### 优点
- 动态大小
- 高效的插入/删除
- 无内存浪费

### 缺点
- 访问速度慢
- 指针需要额外内存
- 缓存局部性差

## 对比表

| 操作 | 数组 | 链表 |
|------|------|------|
| 访问 | O(1) | O(n) |
| 插入 | O(n) | O(1)* |
| 删除 | O(n) | O(1)* |
| 内存 | n | n + 指针 |

*假设位置已找到

## 何时使用

- **数组**: 需要快速随机访问时
- **链表**: 频繁插入/删除时

## 总结

两种数据结构各有用武之地。在需要随机访问时使用数组，在频繁修改时使用链表。
